Relatório de Vulnerabilidades

Vulnerabilidade Encontrada

Detalhes: (1/1) * Possible vulnerability detected: Insecure Random Number Generator

Código Vulnerável:

import java.util.Random;

Sugestão de Correção:

A vulnerabilidade que você encontrou é relacionada ao uso de um gerador de números aleatórios inseguro em Java. A classe java.util.Random pode ser vulnerável a ataques de predição, especialmente em aplicações que necessitam de segurança, tais como geração de senhas, tokens ou chaves criptográficas. Para corrigir essa vulnerabilidade, você deve utilizar um gerador de números aleatórios mais seguro, como java.security.SecureRandom.


Aqui estão os passos para corrigir a vulnerabilidade:


    

  1. 

    Substitua a importação: Troque import java.util.Random; por import java.security.SecureRandom;.
    

    2. 

  2. 

    Instancie SecureRandom: Em vez de criar uma instância de Random, crie uma instância de SecureRandom.
    

    3. 

  3. 

    Modifique seu código para usar SecureRandom: Ajuste como você gera números aleatórios. O SecureRandom funciona de maneira semelhante ao Random, mas fornece uma maior segurança.
    

    4. 



Exemplo:


Antes da correção:
```java
import java.util.Random;


public class App {
    public static void main(String[] args) {
        Random random = new Random();
        int randomNumber = random.nextInt(100); // Gera um número aleatório entre 0 e 99
        System.out.println(randomNumber);
    }
}
```


Depois da correção:
```java
import java.security.SecureRandom;


public class App {
    public static void main(String[] args) {
        SecureRandom secureRandom = new SecureRandom();
        int randomNumber = secureRandom.nextInt(100); // Gera um número aleatório entre 0 e 99 de forma segura
        System.out.println(randomNumber);
    }
}
```


Considerações Finais:


    

  • O SecureRandom pode ter um desempenho ligeiramente inferior ao Random devido à sua natureza mais segura, mas é crucial para aplicações sensíveis à segurança.
    • 

  • Certifique-se de testar seu código após fazer essa substituição, para garantir que ele continue funcionando conforme esperado.
    • 

  • Sempre revise outras partes do seu código que possam estar usando o java.util.Random e considere a mesma substituição.
    • 



Seguindo essas etapas, você deve ser capaz de mitigar a vulnerabilidade reportada pelo Horusec.
